Successful planning is more than just tracking issues

The way a software project is managed can be the difference between success and failure. Software project management methodologies based on agile practices have become very popular alternatives to the old waterfall approach because of the need to release more quickly and more often to provide continuous value.

Many projects use a simple issue tracker, to manage features that need to be implemented and bugs that need to be fixed. While issue trackers might work for fairly small projects, they typically lack the ability to schedule tasks into development iterations and don’t provide a meaningful overview of what’s in progress and what the backlog is. Project managers of well-run projects will add or change management tools to make up for the missing functionality.

While project managers and business analysts adopt more capable management tools, developers tend to stick with the issue tracker, particularly if it’s integrated with their source control system or IDE, When this split happens, it creates a burden for everyone to keep both tools up-to-date to project an accurate view of project status.

No one management methodology is right for every team or every project. Some people prefer the prescriptive nature of formal Scrum processes. Others prefer a less rigid agile approach or something quality driven that follows the Kano model.

What is needed are planning tools that can be used by the whole team including developers and project managers. The tools should be integrated with the development platform and should have the flexibility so you can choose a management methodology that is appropriate for the project.

Red Hat OpenShift.io, an end-to-end development experience

To help development teams be productive Red Hat created OpenShift.io. OpenShift.io is an end-to-end development environment for planning, building and deploying cloud-native applications.

Planning is an integral part of the OpenShift.io team development experience. With OpenShift.io, you choose how you track and prioritize work to be done--all from the same consistent environment. When you create a space for your project on OpenShift.io, you choose your methodology: agile, Scrum™, simple issue tracker, or a new model we are excited about--scenario-driven planning.

You can use task or Kanban boards to get an overview of your project, review the backlog, plan the next development iteration, see work in progress, or just view the work items assigned to you, all from within OpenShift.io in a browser on any device.
